# 产品需求文档（PRD）

## 攒股收息 App

**「被动收入进度条」—— 让每一分红利都看得见、数得着**

版本：V1.0 · 产品阶段：MVP 规划
日期：2026年3月

---

## 1. 背景与问题定义

### 1.1 市场背景

国内 A 股市场有一批以「买高股息、长期持有、靠分红积累被动收入」为核心策略的投资者——这类用户正在快速增长，但市场上没有一款产品真正服务于他们的核心需求。

目前主流的投资类 App（同花顺、东方财富、雪球、华泰涨乐等）产品逻辑均以「交易」为核心：高频行情、快速下单、买卖提醒。这类产品的目标是让用户频繁打开、频繁操作。

「攒股收息」类用户的心智恰恰相反——
- 他们不想每天看行情涨跌
- 他们想知道「我的钱有没有在稳稳地复利增长」
- 他们关心「下一笔分红什么时候到账、到账多少」
- 他们有一个清晰的财务目标：「每月被动收入达到 X 元」

**核心洞察：** 现有工具把用户当「交易者」，没有人把用户当「积累者」。「攒股收息」用户需要的不是行情工具，而是一个「被动收入进度条」——它告诉我：我现在在哪里、我离目标还差多远、我每一步走得对不对。

### 1.2 用户痛点

| 痛点 | 现状 | 影响 |
|------|------|------|
| 无法追踪未来股息 | 分红信息分散在各大券商 App 和公告里，缺乏聚合 | 用户不知道「下个月有多少钱到账」 |
| 缺乏目标感 | 没有任何工具支持「被动收入目标」设定与进度追踪 | 积累过程枯燥、缺乏正向反馈 |
| 多账户数据割裂 | A 股 / 港股 / 美股分散在不同券商，需手动加总 | 无法看到完整持仓和总收息情况 |
| 持仓录入麻烦 | 手动输入成本高，无法与券商直连 | 用户放弃记录 |
| 没有复利可视化 | 无法模拟「若股息全部再投入，未来现金流如何」 | 用户低估了长期复利的价值 |

## 2. 产品定位

### 2.1 一句话描述

> 「帮助 A 股高股息投资者可视化被动收入积累进度，把枯燥的等待变成有成就感的长期旅程。」

### 2.2 核心用户画像

| 维度 | 描述 |
|------|------|
| 人群标签 | 25-45 岁，有稳定收入，持有 A 股高股息个股或指数基金 |
| 投资策略 | 长期持有，不频繁交易，看重分红收益，有一定财务自由目标意识 |
| 关键诉求 | 清楚知道「我每年/每月能收多少分红」「离目标还差多远」 |
| 使用频率 | 非高频：重要节点打开（除息日前后、季报分红公告时） |
| 技术水平 | 普通用户，不需要复杂操作，重视界面清晰直观 |
| 核心情绪 | 追求「安心感」「掌控感」「积累的成就感」，而非刺激感 |

## 3. 目标与成功指标

### 3.1 V1 产品目标

- 验证核心假设：「目标 & 进度系统」是否能成为用户持续回访的驱动力

### 3.2 北极星指标

**次月留存率（Month-1 Retention）**

### 3.3 辅助指标

| 指标 | 目标值（V1 上线后 3 个月） |
|------|--------------------------|
| 持仓录入完成率 | > 60% |
| 目标设置率 | > 40% |
| 股息日历打开率 | > 50% / 周 |

## 5. 功能详述（V1）

### 5.1 模块一：持仓管理

**核心目标：** 让用户以最低成本把自己的 A 股持仓录入系统。

**录入方式：**
1. **CSV 导入** - 用户从券商导出持仓 CSV 文件，App 解析后一键导入
2. **截图 OCR 识别** - 用户截图券商 App 持仓页，OCR 自动识别
3. **手动添加** - 逐支输入股票代码、持仓数量、持仓成本

**持仓卡片展示：**
- 股票名称 + 代码 + 市场标识（A 股）
- 持股数量 + 最新市值（实时行情）
- 历史分红金额（近 3 年）+ 股息率
- 预计下一次分红时间与金额

**V1 约束：** 仅支持 A 股主板、中小板、创业板。

### 5.2 模块二：股息日历

**核心目标：** 让用户一眼看清未来 12 个月哪几天有分红到账、金额是多少。

**日历视图：**
- 以月为单位展示，每个有分红的日期标记高亮
- 点击日期展开：持股名称、每股分红、到账总金额
- 支持「年视图」总览

**状态区分：** 「预案」（橙色） / 「已确认」（绿色） / 「已到账」（蓝色）

**推送通知：**
- 除息日前 3 天提醒
- 分红实际到账提醒

### 5.3 模块三：目标与进度系统（核心体验）

**目标设置：** 用户设定「月度被动收入目标」

**进度展示要求：**
- 进度条是首页最核心的视觉元素
- 数字要大、颜色要暖、要有具体感
- 当前月均收息 / 目标 / 进度% / 还差金额 / 预计达成时间

**里程碑系统：**

| 里程碑 | 触发条件 | 反馈形式 |
|--------|----------|----------|
| 第一笔分红到账 | 首次有分红记录入账 | 动画庆祝 + 纪念卡片 |
| 月收息破千 | 月均收息突破 1000 元 | 进度飞跃提示 |
| 目标达成 25%/50%/75% | 进度跨越阶段节点 | 阶段徽章 + 激励文案 |
| 持股满一年 | 某支股票持有满 365 天 | 「老朋友」标记 |
| 目标达成 | 月均收息 >= 设定目标 | 全屏庆祝动画 + 分享卡片 |

**情绪化设计要点：**
- 用「收到」代替「收益」
- 进度不因股价涨跌而大幅波动（只依赖分红数据）

### 5.4 模块四：会员体系

| 功能模块 | 免费版 | 会员版 |
|----------|--------|--------|
| 持仓数量上限 | 最多 10 支 | 无限 |
| 持仓录入方式 | 手动录入 + 1 次 CSV/月 | 无限次 CSV + OCR |
| 股息日历 | 未来 3 个月 | 未来 12 个月 + 历史 3 年 |
| 目标与进度系统 | 1 个目标 | 多目标 |
| 复利模拟器 | 不可用 | 全功能 |
| 分红推送通知 | 除息日提醒 | 全部推送 |
| 数据导出 | 不支持 | 支持 Excel/PDF |
| 广告 | 展示 | 无广告 |

**定价：** 月度 18 元/月，年度 98 元/年

## 6. 数据与技术约束

### 6.1 A 股股息数据源

| 数据类型 | 推荐数据源 |
|----------|-----------|
| 分红预案/公告 | 上交所、深交所官方公告接口 |
| 除权除息日 | Tushare Pro / AKShare |
| 实际到账日 | 推算（除息日后 1-5 个工作日） |
| 实时股价 | 第三方行情数据服务 |

### 6.2 技术架构

| 端 | 技术选型 |
|----|---------|
| Web 端 | Next.js (App Router) + Tailwind CSS |
| 后端 | Node.js + Prisma |
| 数据库 | SQLite (MVP) → PostgreSQL |
